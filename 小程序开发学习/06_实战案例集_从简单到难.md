## 🎯 案例 1:Hello World + 个人简介 ⭐

### 目标
开发者工具新建项目,修改默认页显示成你的"个人名片"。

### 需要的技能
- 微信开发者工具基础
- WXML / WXSS 基础

### 步骤
1. 新建项目,选 JS 基础模板
2. 打开 `pages/index/index.wxml` 把所有内容换成:
   - 你的头像(用本地图片)
   - 名字
   - 一句简介
3. 在 `index.wxss` 加样式(居中、颜色、圆头像)
4. 手机预览

### 能学到
- 熟悉开发者工具布局
- 理解 4 个文件的作用
- 第一次"跑通"小程序

### AI 辅助 Prompt
> "给我一个个人名片小程序的 WXML + WXSS 代码,要求头像圆形居中、名字大字、底下一行简介,风格简洁"

## 🎯 案例 2:计数器 ⭐

### 目标
点 "+" 加 1,点 "-" 减 1,显示数字。

### 需要的技能
- setData
- 事件绑定

### 步骤
1. data 里存 `count: 0`
2. WXML 两个按钮 + 显示数字
3. JS 写 onAdd / onMinus 方法

### 能学到
- **setData 的用法(最核心)**
- 事件绑定
- 数据驱动视图

### AI Prompt
> "帮我写一个小程序计数器的完整 4 个文件代码,要有 +/- 按钮和当前数字"

---

## 🎯 案例 3:待办清单 TodoList ⭐⭐

### 目标
- 输入任务 → 添加
- 点任务 → 标记完成(打勾)
- 长按 → 删除
- 数据存本地(关闭再打开还在)

### 需要的技能
- wx:for 列表渲染
- wx:if 条件渲染
- wx.setStorageSync 本地存储

### 步骤
1. 输入框 + 添加按钮
2. 列表展示(wx:for)
3. 点击切换完成状态
4. 长按弹出删除确认
5. 每次改动存到本地

### 能学到
- **列表的增删改查**
- 本地存储
- 完整的单页面应用

### AI Prompt
> "帮我做一个小程序 TodoList,要有添加、完成、删除、本地持久化,给我完整代码"

## 🎯 案例 4:多页面笑话 App ⭐⭐

### 目标
- 首页:展示笑话列表(3 个 tab:冷笑话、网络笑话、脑筋急转弯)

3. 点击跳转详情,传递 id

### 能学到

- 路由和传参
- tabBar 的用法

### AI Prompt
> "帮我规划一个笑话小程序,有首页(带 tab)、详情页、收藏页,app.json 怎么配?"

## 🎯 案例 5:天气小程序 ⭐⭐⭐

### 目标
- 用户输入城市 → 查询天气
- 显示当天 + 未来 7 天
- 调用第三方天气 API

### 需要的技能
- wx.request 网络请求
- Promise / async/await
- JSON 数据处理

### 步骤
1. 注册免费天气 API(心知、和风天气)
2. 输入框 + 查询按钮
3. wx.request 调接口
4. 渲染返回的 JSON 数据

### 能学到
- **接口调用(非常重要)**
- 异步处理
- 真实数据展示

### 坑
- 小程序要求 HTTPS + **域名白名单**(开发时可以在开发者工具关闭校验)

### AI Prompt
> "帮我写一个小程序请求心知天气 API 的代码,返回数据后渲染到界面"

## 🎯 案例 9:小程序 + 支付(企业主体)⭐⭐⭐⭐⭐

### 目标
给案例 8 的点餐小程序,加**真实微信支付**。

### 前置条件
- **必须企业主体**(个人主体无支付)
- 商户号 + 认证

### 步骤
1. 申请小程序认证(300 元)
2. 开通微信支付,拿商户号
3. 云函数调预下单接口
4. 前端 wx.requestPayment
5. 云函数验证支付回调

### 能学到
- **支付全流程**
- 订单状态管理
- 财务对账基础

## 💡 实战通用心法

### 1. **从小做到大**
别一开始就做微信克隆版。先做 Hello World,再一点点加功能。

### 2. **同一个项目做 3 遍**
第一遍能跑就行,第二遍优化代码,第三遍加新功能。

### 3. **把每个案例上传到 GitHub**
将来找工作 / 接单,就是你的作品集。

### 4. **记录所有报错**
建一个"错误库"文档,以后能秒查。

### 5. **发给朋友用**
哪怕是 Demo,也要让真人用一下,收到真实反馈才知道哪里有问题。
