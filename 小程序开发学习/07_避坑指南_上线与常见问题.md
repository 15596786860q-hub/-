## 🚨 一、小程序审核红线(务必看完)

### 绝对过不了审的 10 种内容 ❌

| 类型 | 示例 | 后果 |
|------|------|------|
| **涉政敏感** | 包含敏感人物、政治话题 | 直接封号 |
| **色情低俗** | 擦边图片、性暗示内容 | 封号 + 上报 |
| **赌博博彩** | 棋牌、老虎机 | 永久封禁 |
| **医疗误导** | 无资质的"养生"、"治病" | 拒审 |
| **金融违规** | 无牌的理财、借贷 | 拒审 |
| **虚假宣传** | "最佳"、"第一"、"100%" | 拒审 |
| **侵权素材** | 盗用其他 App 的图标、截图 | 拒审 |
| **诱导分享** | "分享才能继续"、"请 10 个朋友" | 警告 + 封禁功能 |
| **超范围经营** | 类目不符的服务 | 拒审 |
| **破坏生态** | 导流到其他平台 | 警告 |

### ⚠️ 经常踩的 5 个坑

#### 坑 1:类目不符
**例**:你申请了"生活服务",但做了"教育培训"功能 → 拒审。
**对策**:写代码前先想清楚类目,注册时选对。

#### 坑 2:没有隐私协议
**从 2023 年开始强制要求**:
- 必须有《用户协议》和《隐私政策》页面
- 首次使用必须弹窗同意
- 要在微信后台"设置" → "服务内容声明" 填

#### 坑 3:功能不完整
审核员打开发现:
- 空页面、404
- 按钮点了没反应
- 登录进不去

**对策**:提审前自己完整走一遍所有功能。

#### 坑 4:测试账号没给
很多功能需要登录、付费才能用,审核员**没账号看不到**。
**对策**:在审核备注里填"测试账号:xxx / 密码:xxx"。

#### 坑 5:内容类但没版号 / 证照
- 做电商 → 需要营业执照 + 食品流通许可证(如果卖食品)
- 做资讯 → 需要 ICP 备案和《互联网新闻信息服务许可证》
- 做游戏 → 需要游戏版号

## 📋 二、上线前完整检查清单

### 代码层

## ⏱ 三、审核流程时间表

```
提交代码 → 上传到后台 → 提交审核 → 审核中
    ↓            ↓            ↓         ↓
   秒级        秒级         1-7 天     通过/驳回
```

- **首次审核**:通常 2-7 天
- **后续迭代**:通常 1-3 天
- **驳回**:修改后重新提交,再等相同时间
- **节假日**:可能延迟

### 加急审核
- 有些情况可申请加急(紧急 bug)
- 在后台点"审核 → 加急"
- 每月有限额

## ❓ 四、常见问题 FAQ

### 基础类

#### Q1:注册小程序需要什么?
**A**:
- **个人主体**:
  - 邮箱(未被注册过)
  - 身份证
  - 微信实名
  - 免费
- **企业主体**:
  - 企业邮箱
  - 营业执照
  - 对公账户
  - 300 元认证费(每年)

#### Q2:个人主体能做哪些功能?
**A**:
- ✅ 基础展示(介绍、联系)
- ✅ 工具类(日历、计算器)
- ✅ 社区类(论坛、留言)
- ❌ 支付收款(必须企业)
- ❌ 直播
- ❌ 卖东西(电商类目)

**新手建议**:个人主体练手,要上线收款换企业。

#### Q3:一个人可以注册几个?
**A**:
- 同一个身份证最多 5 个个人小程序
- 同一个企业最多 50 个小程序

#### Q4:小程序名字怎么起?
**A**:
- 4-30 字符
- 每年能改 2 次(个人) / 4 次(企业)
- 不能和现有的重名
- 别用"第一"、"最"等宣传词

#### Q5:可以卖货吗?
**A**:
- 个人主体:❌ 不能(没支付)
- 企业主体:✅ 可以,但要选对类目(电商/零售)

---

### 开发类

#### Q6:本地开发好好的,真机闪退?
**A**:常见原因:
1. 用了真机不支持的 API
2. 图片路径写错(开发工具不区分大小写,真机区分)
3. setData 传太大的数据
4. 安卓某些版本兼容性问题

**对策**:
- 开发时经常真机预览
- 开发者工具里开"详情 → 本地设置 → 不校验合法域名"只在开发用,真机关掉

#### Q7:wx.request 报"不在以下 request 合法域名列表中"?
**A**:
- 上线版本:去小程序后台"开发 → 开发管理 → 开发设置 → 服务器域名"配置
- 要求:HTTPS + 已备案的域名
- 开发时:可在开发者工具勾选"不校验合法域名"跳过

#### Q8:setData 有什么性能陷阱?
**A**:
- ❌ 频繁调用(循环里调)
- ❌ 传大对象
- ❌ 传整个 data
- ✅ 批量更新:一次 setData 传多个字段
- ✅ 局部更新:`setData({ 'list[0].name': 'xxx' })`

#### Q9:分包是什么?什么时候要用?
**A**:
- 微信小程序主包限制 2MB
- 超过要拆分成"主包 + 子包"
- 每个子包 2MB,总共 20MB
- 拆法:不常用的页面放子包,用户访问时才下载

#### Q10:小程序和 H5 能互相跳转吗?
**A**:
- 小程序 → H5:用 `web-view` 组件打开(需要业务域名配置)
- H5 → 小程序:用 "url-scheme" 或"微信内分享的链接"
- 小程序之间:`wx.navigateToMiniProgram`

### 云开发类

#### Q11:云开发免费额度够用吗?
**A**:
- 新用户赠送 1 个月资源
- 日常:
  - 云函数 40 万次调用/月
  - 云数据库 2GB 存储、5 万次读、1 万次写/天
  - 云存储 5GB
- **个人小项目完全够用**
- 超出按量付费,月几十元

#### Q12:云数据库怎么做关联查询?
**A**:
- 云数据库是 NoSQL,不像 MySQL 那么强
- 简单关联:`lookup`(类似 join)
- 复杂关联:拆成多次查询,在云函数里组合
- 学习资源:云开发官方文档

#### Q13:一个云开发环境能用多久?
**A**:
- 环境不用会自动冻结,但**数据不丢**
- 解冻只要重新开通
- 企业建议直接买固定套餐

### 上线类

#### Q14:第一次提审被拒怎么办?
**A**:
- **别慌**,90% 的小程序都被拒过
- 仔细读驳回理由
- 对照整改,再提审
- 一般第 2-3 次能过

#### Q15:被拒的常见理由 TOP 5
1. 类目错误 / 功能不符
2. 存在 bug / 空页面
3. 诱导关注 / 分享
4. 涉及敏感内容
5. 隐私协议问题

#### Q16:上线后可以改代码吗?
**A**:
- **可以**,每次改都要重新审核
- 紧急 bug 可申请加急
- 用"灰度发布"功能先发小部分用户试

### 商业类

#### Q17:接单价格怎么定?
**A**(2025-2026 行情参考):
- 企业展示型:3000-8000
- 点餐 / 预约:5000-15000
- 商城:15000-50000
- 复杂业务:50000+

新手前 3 单可以便宜点(3000-5000),**重要的是积累作品集**。

#### Q18:怎么找客户?
**A**:
- 闲鱼、小红书、抖音发案例
- 朋友圈 + 行业群
- 接包平台:程序员客栈、码市
- 最稳:身边的餐饮、美容店老板

#### Q19:自己做产品能赚钱吗?
**A**:难,但有可能:
- 工具类:流量主 + 会员
- 内容类:知识付费
- 电商:卖货分成
- 服务类:佣金

**建议**:先接单赚生活费,副业时间做自己产品。

#### Q20:要不要买课?
**A**:
- **免费资源已经够了**:B 站、官方文档、AI
- 买课的价值:**省时间 + 有老师答疑**
- 推荐(不是广告):
  - **微信官方文档**(最权威)
  - **B 站 CodeWithMosh 的小程序教程**
  - **掘金小册**(有付费文章)

## 🚫 五、新手最容易踩的 15 个坑

### 开发坑

1. **不看官方文档就乱猜 API** → 永远以官方文档为准
2. **开发工具"不校验域名"上线了** → 上线前要关掉
3. **图片用本地路径,真机路径错** → 用绝对路径或 cloud://
4. **数据更新不用 setData** → this.data.xx = 不会更新界面
5. **频繁 setData 导致卡顿** → 合并调用
6. **没有统一的请求封装** → 每个页面都 wx.request 是灾难
7. **console.log 满天飞上线** → 拖累性能

### 上线坑

8. **类目选错,功能写了白做** → 注册前就想清楚
9. **没隐私协议页** → 直接拒审
10. **首次使用没弹同意** → 直接拒审
11. **审核备注没给测试账号** → 审核员看不到核心功能
12. **改了小程序名字不能搜到** → 名字要考虑 SEO
13. **域名没备案就用** → 线上必定 500

### 商业坑

14. **做全套就烂尾** → 先 MVP
15. **无差异化产品** → 小红书已经有了,你做第二个没人用

## 📜 七、合规建议

### 做商业项目必须要的

1. **企业认证**(300/年)
2. **ICP 备案**(免费但要 2-3 周)
3. **经营许可证**(根据类目,如食品经营、教培资质)
4. **隐私政策 / 用户协议**(模板网上能找)
5. **发票管理**(卖货类)

### 个人开发者建议

1. 先做**免费工具类**(练手、无需资质)
2. 想商业化 → 注册个体工商户(简单便宜)→ 升级企业主体
3. 别碰医疗、金融、教育这种高压线类目

## 💎 八、给你的 10 条"黄金建议"

1. **别等完美再上线**,"能用"就先发
2. **每天写代码 > 周末狂学**
3. **先会原生,再学 Taro/Vue**,别反着来
4. **把每个 Bug 记下来**,形成自己的错误库
5. **写代码前先画原型**(哪怕在纸上)
6. **先学前端 80% 再学后端 20%**(云开发降低门槛)
7. **加入社区,不要孤军奋战**
8. **你的第一个小程序一定丑**,不重要
9. **接单前,一定要有 3 个上线作品**
10. **赚到钱的一天 = 你真的学会了的一天**

## 🎯 九、最后的成长路径图

```
第 1 月:学基础 (HTML/CSS/JS)
      ↓
第 2 月:小程序入门(做出能跑的 Demo)
      ↓

```
