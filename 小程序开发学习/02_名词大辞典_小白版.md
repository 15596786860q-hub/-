## A

### AppID
**一句话**:小程序的**身份证号**,每个小程序独一无二。
**怎么拿**:注册小程序账号后,在公众平台后台"开发管理"里能看到。
**例**:`wx1234567890abcdef`

### API(Application Programming Interface)
**一句话**:"接口",是你调用别人能力的**入口**。
**类比**:餐厅的**菜单** —— 你点菜(调 API),后厨给你做(系统执行),不用关心怎么做的。
**例子**:
- `wx.login()` → 微信登录 API
- `wx.request()` → 发请求 API
- `wx.getLocation()` → 获取位置 API

### async / await
**一句话**:JavaScript 里**等一件事做完再做下一件**的优雅写法。
**为什么要学**:发请求、读数据库都是"要等"的事,用 async/await 代码更清晰。
**例**:
```js
async function getUser() {
  const res = await wx.login();  // 等 login 完成
  console.log(res.code);          // 再执行这行
}
```

## B

### Babel
**一句话**:把新语法(ES6+)翻译成旧浏览器也能懂的 JS。
**小程序中**:开发者工具自带,不用操心。

### BaaS(Backend as a Service)
**一句话**:把后端"外包"给服务商,你不用自己搭服务器。
**例子**:微信云开发、LeanCloud、Bmob。
**好处**:新手省心,不用学后端也能做完整项目。

### Bind / Catch
**一句话**:小程序里的**事件绑定**方式。
- `bindtap` = 点击事件(会冒泡)
- `catchtap` = 点击事件(不冒泡,阻止事件往上传)

### BoundingClientRect
**一句话**:获取一个元素的**位置和大小**。
**什么时候用**:做滚动、动画、自定义定位。

---

## C

### Canvas
**一句话**:小程序里的**画布**,能画图、做图表、生成海报。
**典型场景**:分享海报、签到日历、小游戏。

### CDN(Content Delivery Network)
**一句话**:把图片/视频分发到全国各地的服务器,**加速访问**。
**类比**:全国连锁店,你去最近的那家就行。

### CI/CD(持续集成/持续部署)
**一句话**:代码写完自动测试、自动上传。
**工具**:微信开发者工具的 miniprogram-ci、GitHub Actions。

### Component(组件)
**一句话**:可以**重复使用**的 UI 小块(按钮、卡片、列表)。
**类比**:乐高积木,拼大作品用的。

### Cookie / Session / Token
- **Cookie**:浏览器存的小数据(小程序没有)
- **Session**:服务器存的会话(传统网页常用)
- **Token**:小程序主流登录凭证,类似"通行证"

### CRUD
**一句话**:数据库的 4 种操作:
- **C**reate(增)
- **R**ead(查)
- **U**pdate(改)
- **D**elete(删)

---

## D

### data(数据)
**一句话**:页面里存数据的**小仓库**。
**例**:
```js
Page({
  data: {
    name: "小明",
    age: 18
  }
})
```
修改数据要用 `setData`,不能直接改。

### DOM
**一句话**:网页的"元素树"。
**小程序不用**:小程序不直接操作 DOM,用 setData 更新。

### debug(调试)
**一句话**:找出代码里的 bug(错误)。
**神器**:`console.log()` 打印变量,**新手最常用**。

---

## E

### ECMAScript(ES)
**一句话**:JavaScript 的**官方标准**。
- ES5 = 旧版(2009)
- ES6 = 新版(2015,大升级)
- ES2020 / ES2022 = 每年更新

### 事件(Event)
**一句话**:用户的**动作**(点击、输入、滑动)。
**小程序里怎么响应**:在 WXML 用 `bindtap`,在 JS 写对应函数。

### ESLint
**一句话**:代码**风格检查器**,帮你揪出不规范的写法。

---

## F

### Flex 布局
**一句话**:CSS 里**最好用的布局方式**。
**小程序里必须会**:上下、左右对齐全靠它。
```css
.container { display: flex; justify-content: center; }
```

### Function(函数)
**一句话**:一段**能重复用**的代码。
**类比**:菜谱 —— 写一次,做多次。

---

## G

### globalData
**一句话**:**全小程序共享**的数据,放在 `app.js` 里。
**例子**:登录用户信息、主题配色。

### Git
**一句话**:代码**版本管理**工具,能保存每次改动,像游戏存档。
**必须学**:不然代码改坏了救不回来。

### GitHub
**一句话**:全球最大的**代码托管平台**,像"程序员的朋友圈"。

### GET / POST
**一句话**:HTTP 请求的两种**常见方式**。
- GET = 获取数据(查列表)
- POST = 提交数据(新增订单)

---

## H

### HTTP / HTTPS
**一句话**:浏览器和服务器**对话的语言**。
- HTTP:不加密,不安全
- HTTPS:加密,**小程序必须用 HTTPS**

### HTML / CSS / JS
前端三兄弟:
- **HTML**:骨架(结构)
- **CSS**:皮肤(样式)
- **JS**:大脑(逻辑)

### Hybrid(混合开发)
**一句话**:App 里嵌一个网页。和小程序不太一样。

---

## I

### IDE(Integrated Development Environment)
**一句话**:写代码的软件。**微信开发者工具**就是小程序的专属 IDE。

### if / else
**一句话**:编程里的**如果...就...否则...**。

### input
**一句话**:输入框组件。

---

## J

### JavaScript(JS)
**一句话**:小程序**最核心的编程语言**。不会 JS 做不了小程序。

### JSON
**一句话**:一种**数据格式**,长这样:
```json
{
  "name": "小明",
  "age": 18,
  "hobbies": ["打球", "看书"]
}
```
**应用**:配置文件、API 返回数据。

### JWT(JSON Web Token)
**一句话**:一种**登录凭证**,前后端都用。
**类比**:电影票 —— 你拿着它就证明你是付过钱的。

---

## K

### Key(键)
**一句话**:
- 在 `wx:for` 列表渲染里,给每项一个**唯一标识**,提升性能。
- 在对象里 `{ name: "小明" }`,`name` 就是 key。

---

## L

### LBS(Location-Based Service)
**一句话**:**基于位置的服务**(附近的 xx、叫外卖)。
**相关 API**:`wx.getLocation`。

### let / const / var
JS 的三种**变量声明**:
- `let`:可以改的变量(推荐)
- `const`:不能改的常量(推荐)
- `var`:老式写法(别用)

### LifeCycle(生命周期)
**一句话**:页面/小程序**从打开到关闭**经历的各个阶段。
**小程序里的生命周期**:
- `onLoad`:页面加载
- `onShow`:页面显示
- `onHide`:页面隐藏
- `onUnload`:页面销毁

## M

### MVP(Minimum Viable Product)
**一句话**:**最小可行产品**,先做个能用的丑版本试水。
**新手必学的产品思维**:先上线,再迭代。

### MobX / Redux
**一句话**:**状态管理**库,用来管理复杂数据。
**小项目不用**,中大项目再考虑。

### mp(小程序)
**一句话**:"miniprogram" 的缩写。微信后台网址就是 `mp.weixin.qq.com`。

---

## N

### Node.js
**一句话**:让 **JavaScript 能在服务器端跑**的运行时。
**你需要它**:装小程序的一些工具、写云函数。

### npm(Node Package Manager)
**一句话**:JS 的**包管理器**,全球最大代码仓库。
**常用命令**:
- `npm install xxx` 装包
- `npm run dev` 跑项目

### NaN
**一句话**:"Not a Number"(不是数字),JS 里的特殊值。
**什么时候出现**:`"abc" * 2` 这种鬼计算。

---

## O

### OAuth
**一句话**:**授权登录协议**(微信登录、QQ 登录都用这个)。

### OpenID
**一句话**:**用户在你小程序里的唯一标识**。
**关键**:
- 同一个用户在**你的小程序**里 OpenID 不变
- 在**别的小程序**里 OpenID 不一样(隐私保护)
- 想要跨小程序识别用户 → 用 **UnionID**

### onLoad / onShow / onReady / onHide / onUnload
小程序**页面生命周期**函数(前面讲过 LifeCycle)。

---

## P

### Page(页面)
**一句话**:小程序的一个"屏幕"。
**结构**:每个页面有 4 个文件(.wxml / .wxss / .js / .json)。

### Promise
**一句话**:处理**异步操作**的工具。
**和 async/await 关系**:async/await 是 Promise 的"好用版"。

### Picker
**一句话**:小程序的**选择器组件**(日期、时间、地区、多列选择)。

### Preview(预览)
**一句话**:开发者工具里"预览"按钮,生成二维码,**手机扫码体验**。

---

## R

### rpx(responsive pixel)
**一句话**:小程序独有的**响应式单位**,自动适配不同屏幕。
**换算**:屏幕宽 = 750rpx(不管手机大小)
**例**:`width: 200rpx;` → 在 iPhone X 约等于 100px。

### require / import
JS 里**引入模块**的两种写法:
- `require('xxx')` 老写法(CommonJS)
- `import xxx from 'xxx'` 新写法(ES Module)
- 小程序两种都能用

### Restful API
**一句话**:一种**设计 API 的规范**,用 HTTP 动词表达操作。
- `GET /users` 获取用户列表
- `POST /users` 创建用户
- `DELETE /users/1` 删除用户 1

---

## S

### setData
**一句话**:小程序里**更新界面数据**的方法。
**关键**:不能直接改 `this.data.xxx`,必须用 setData。
```js
this.setData({ name: "新名字" })
```

### Server(服务器)
**一句话**:存数据、跑代码的**远程电脑**。
**小程序要**:HTTPS 域名、备案、SSL 证书。

### Slot(插槽)
**一句话**:自定义组件里的**占位符**,让用户塞自己的内容。

### SPA(Single Page Application)
**一句话**:**单页应用**,整个网站就一个 html 页面。
**小程序**:结构类似 SPA,但更规整。

### SSL
**一句话**:HTTPS 需要的**加密证书**。
**怎么拿**:阿里云 / 腾讯云免费申请。

### state / props
组件通信的两个核心:
- **state**:组件**自己的**数据
- **props**:**爹传给**组件的数据

---

## T

### tabBar
**一句话**:小程序底部的**导航栏**(首页/消息/我的)。
**在哪配**:`app.json` 里。

### TypeScript(TS)
**一句话**:**加了类型的 JavaScript**。
**新手先学 JS,后期再学 TS**。

### Template(模板)
**一句话**:可复用的 WXML 片段。
**现在建议用 Component 替代**。

### Toast
**一句话**:屏幕中央的**小提示框**(成功/失败)。
```js
wx.showToast({ title: "登录成功" })
```

---

## U

### UI(User Interface)
**一句话**:**用户界面**,用户看到的一切。

### UX(User Experience)
**一句话**:**用户体验**,用起来爽不爽。

### URL
**一句话**:**网址**。

### UnionID
**一句话**:和 OpenID 类似,但**同一个用户在同主体的多个小程序/公众号里是同一个**。
**用途**:多个小程序打通账号。

---

## V

### Vant Weapp / TDesign
**一句话**:现成的**小程序 UI 组件库**,拿来就能用。
**好处**:节省 80% UI 开发时间。

### Vue / React
两个**主流前端框架**:
- **Vue**:国产神器,语法简单
- **React**:Facebook 出品,生态最大
- 小程序原生不用这俩,但 Taro/Uni-app 会用到

---

## W

### WXML(WeiXin Markup Language)
**一句话**:小程序的**模板语言**,类似 HTML。
```html
<view>{{name}}</view>
<button bindtap="onClick">点我</button>
```

### WXSS(WeiXin Style Sheets)
**一句话**:小程序的**样式语言**,类似 CSS。
**区别**:有 rpx 单位、不支持某些 CSS3 特性。

### WebSocket
**一句话**:**双向通信**协议,服务器也能主动给客户端发消息。
**应用**:聊天、实时推送、游戏。

### 微信开发者工具
**一句话**:小程序**官方 IDE**,调试、编译、上传一条龙。

### 微信云开发(CloudBase)
**一句话**:**不用服务器**就能写后端的方案。
**包含**:云函数、云数据库、云存储。

## 中文名词

### 冒泡
**一句话**:事件从**里层向外层传递**。
**例**:你点按钮,按钮触发 → 外层 view 也触发。
**阻止冒泡**:用 `catchtap` 代替 `bindtap`。

### 主体
**一句话**:小程序注册时填的**运营者身份**。
- **个人主体**:个人名字,没支付等高级功能
- **企业主体**:公司营业执照,功能全

### 备案
**一句话**:**用于小程序的服务器域名**必须向工信部备案。
**流程**:阿里云 / 腾讯云一键申请,约 2-3 周。

### 分包
**一句话**:把小程序拆成**主包 + 子包**,用户点到某功能才下载对应代码。
**限制**:微信小程序主包最大 2MB,总大小最多 20MB。

### 插件
**一句话**:别人开发好的**功能模块**,你引用就能用。
**例**:地图插件、客服插件。

### 生命周期
**一句话**:页面 / 小程序**从生到死的各个阶段**。
- 应用生命周期:`onLaunch`、`onShow`、`onHide`
- 页面生命周期:`onLoad`、`onShow`、`onHide`、`onUnload`

### 审核
**一句话**:你提交代码后,**微信官方人工 + AI 检查**,通过了才能发布。
**时间**:通常 1-3 天,复杂的 1 周。

### 运营者
**一句话**:**负责管理这个小程序的人**(可以和开发者不同)。

### 订阅消息
**一句话**:用户同意后,**你能推送消息到他的服务通知**。
**替代**:旧的模板消息。

### 云函数
**一句话**:写在云端的 JavaScript 函数,前端直接调。
**好处**:不用管服务器、自动扩容。

### 云数据库
**一句话**:微信提供的**JSON 风格数据库**(类 MongoDB)。
**好处**:前端直接读写(带权限控制),不用写后端。

### 真机调试
**一句话**:把代码推到**真手机**上跑,发现模拟器发现不了的问题。

### 分享 / 转发
- **转发**:分享给**微信好友 / 群**
- **分享到朋友圈**:发到朋友圈(需要特殊授权)

### 插件 / 插件市场
**一句话**:第三方开发的模块,能集成到你的小程序里(审核是双方的)。

### 热更新
**一句话**:用户打开小程序,**自动拉取最新版本**。
**区别**:App 要用户手动下载更新。

### 沙箱
**一句话**:小程序跑在**受限的环境**里,不能乱动用户手机(保护隐私和安全)。

### 模板消息(已废弃)
**一句话**:老版本的推送方式,现在用**订阅消息**。
