
## 二、核心架构:**双线程** ⭐ 必须理解

小程序和普通网页**最大的区别**:渲染和逻辑分开跑在两个线程里。

```
┌─────────────────────────────────────────────────┐
│             【用户的手机】                        │
│                                                 │
│  ┌───────────────┐   通信桥    ┌──────────────┐ │
│  │   渲染层       │◄─────────►│   逻辑层      │ │
│  │ (View Thread) │ JSBridge  │ (App Service) │ │
│  │               │           │               │ │
│  │ WXML + WXSS   │           │ JavaScript    │ │
│  │ 负责画界面     │           │ 处理数据逻辑   │ │
│  └───────────────┘           └──────────────┘ │
│         ▲                           ▲           │
│         └────────┬──────────────────┘           │
│                  ▼                              │
│          微信 Native 层                          │
│   (支付/位置/相机/存储/蓝牙/扫码...)              │
└─────────────────────────────────────────────────┘
                  │
                  │ HTTPS / WSS
                  ▼
        ┌──────────────────┐
        │     服务端       │
        │ (云函数/自建后端) │
        └──────────────────┘
```

### 🧠 用大白话解释

| 角色 | 干什么 | 类比 |
|------|-------|------|
| **渲染层** | 画界面,用户看到的按钮、文字 | 店员(展示菜单) |
| **逻辑层** | 跑 JS 代码,处理数据和业务 | 后厨(做饭) |
| **JSBridge** | 两边通信的桥 | 传菜口 |
| **微信 Native** | 支付、定位等原生能力 | 供货商 |
| **服务端** | 云端接口和数据库 | 公司总部 |

**关键**:渲染和逻辑**物理隔离** → 用户点 10 个按钮,界面不会卡 → 体验好。

## 三、金字塔分层:从小白到高手的 5 层 🏔

```
                       ┌─────────────────────────┐
                       │   第 5 层  架构师级       │  ← 大神
                       │  (性能/多端/微服务)     │
                       ├─────────────────────────┤
                       │   第 4 层  后端 / 云开发  │  ← 全栈
                       │   (数据库/接口/支付)     │
                       ├─────────────────────────┤
                       │   第 3 层  进阶前端       │  ← 核心
                       │   (组件化/状态/路由)     │
                       ├─────────────────────────┤
                       │   第 2 层  基础前端       │  ← 入门
                       │   (WXML+WXSS+JS+API)    │
                       ├─────────────────────────┤
                       │   第 1 层  编程基础       │  ← 地基
                       │   (HTML/CSS/JavaScript) │
                       └─────────────────────────┘
```

### 每一层是什么?一句话解释

| 层级 | 名字 | 一句话 | 对应目标 |
|------|------|-------|---------|
| **1** | 编程基础 | 学 JS 的基本语法和概念 | 能写 if、for、函数 |
| **2** | 基础前端 | 学小程序的模板和 API | 做出单页面的 UI |
| **3** | 进阶前端 | 学组件、路由、状态管理 | 做多页面复杂应用 |
| **4** | 后端 / 云开发 | 学数据库、接口、支付 | 做真正能赚钱的产品 |
| **5** | 架构师级 | 性能优化、多端、微服务 | 带团队、大项目 |

## 五、三种主流**开发模式**

### 套路 A:纯原生 + 云开发 ⭐ 新手首选
**步骤**:开发者工具 → 原生代码 → 云函数 + 云数据库 → 发布
**适合**:个人开发者、MVP、学习者
**优点**:不用服务器、省钱、快
**缺点**:只能发微信小程序

### 套路 B:原生 + 自建后端
**步骤**:开发者工具 → 原生代码 → 调自己的 Node/Java/Python 接口
**适合**:团队项目、已有后端
**优点**:灵活、可控
**缺点**:需要服务器、备案

### 套路 C:跨端框架(Taro / Uni-app)
**步骤**:React/Vue 写一次 → 编译成微信/支付宝/抖音/H5 等
**适合**:要多端上架的项目
**优点**:一套代码多端复用
**缺点**:学习曲线陡、调试复杂

> 💡 **学习顺序建议**:A(打基础)→ 会了再考虑 B/C。别一开始就冲 Taro。

## 九、小程序的 3 种形态

```
┌────────────────────────────────────────────┐
│ 1. 小程序                                   │
│    完整的应用,有底部 tab、多页面            │
│    例:京东、美团外卖                        │
├────────────────────────────────────────────┤
│ 2. 小游戏                                   │
│    基于 Canvas 的游戏                       │
│    例:跳一跳、羊了个羊                     │
├────────────────────────────────────────────┤
│ 3. 小程序·服务号插件 / 公众号菜单              │
│    嵌入其他生态的入口                        │
│    例:公众号底部菜单里的"在线咨询"          │
└────────────────────────────────────────────┘
```

**这份文档主要讲第 1 种(应用型小程序)**,这是最主流的。

## 十、生态对比:微信 vs 其他小程序

| 平台 | 用户量 | 主要场景 | 开发难度 |
|------|-------|---------|---------|
| **微信小程序** | 9 亿 | 社交/工具/电商 | 低 |
| **抖音小程序** | 7 亿 | 视频带货 | 中 |
| **支付宝小程序** | 8 亿 | 金融/生活服务 | 中 |
| **百度智能小程序** | 3 亿 | 搜索/工具 | 中 |
| **快手小程序** | 4 亿 | 短视频电商 | 中 |
| **QQ 小程序** | 5 亿 | 年轻用户 | 低 |

**结论**:**先学微信小程序**,语法相似,其他平台容易迁移。
