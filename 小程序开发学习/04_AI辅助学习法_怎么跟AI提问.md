# 04 | AI 辅助学习法:怎么跟 AI 高效对话

> **这是最重要的一篇文档。**
> 学会用 AI,你的学习速度能 **快 10 倍**。学不会用 AI,有神仙老师也没用。

---

## 一、AI 能给你什么?不能给你什么?

### ✅ AI 擅长做的
- 📖 **解释任何概念**(大白话、带类比)
- 🔧 **生成代码**(WXML 页面、JS 函数、云函数)
- 🐛 **调试报错**(粘贴错误,90% 能解决)
- 🧠 **代码审查**(帮你找 bug、优化写法)
- 🎨 **改 UI 样式**(一句话让界面变好看)
- 📝 **总结知识点**(学完一块让它帮你总结)
- 🎯 **出练习题**(让它给你出题验证学习效果)
- 💡 **产品建议**(做什么产品、怎么设计功能)

### ❌ AI 不擅长做的
- 🙅 **替你操作**(安装工具、点按钮 —— 必须你自己动手)
- 🙅 **保证绝对正确**(可能有"幻觉",关键结果自己验证)
- 🙅 **代替思考**(你不动脑,就是"喂答案")
- 🙅 **懂你的产品**(你得讲清楚需求)

## 二、核心心法 ⭐⭐⭐

### 🎯 心法 1:**学 → 做 → 问 → 再做**

```
      ┌─ 学(看文档 / 让 AI 讲)
      │
      ▼
    做一下(动手写代码)
      │
      ▼
   遇到问题 ──→ 问 AI
      │              │
      │    ←─────────┘
      ▼
   再做(用 AI 给的思路)
      │
      ▼
    学会了 ✓
```

**最致命的错误**:看了就算学了。**必须动手**。

### 🎯 心法 2:**问具体的,不问抽象的**

❌ 烂问题:"怎么做小程序?"(太宽)
✅ 好问题:"我想做一个点餐小程序,第一步 UI 怎么画?给我一个简单的首页 WXML 示例"

### 🎯 心法 3:**给 AI 足够的上下文**

AI 不知道你的:
- 操作系统(Win? Mac?)
- 技术栈(用原生还是 Taro?)
- 现在学到哪一步
- 之前做过什么

**每次问,先交代清楚。**

### 🎯 心法 4:**AI 答不对,换种问法**

如果 AI 给的答案不对/听不懂,可以说:
- "我是零基础小白,能用更简单的话解释吗?"
- "能给个生活类比吗?"
- "这个解释里 XX 我不懂,单独展开讲"
- "这段代码运行报错 xxx,帮我修"

## 三、10 个神级 Prompt 模板(复制即用)

### 📝 模板 1:概念解释器

```
你是一位资深的小程序开发讲师,现在我是零基础小白。

请用大白话解释下面这个概念:
【概念】:__________

要求:
1. 一句话核心定义
2. 一个生活化类比
3. 在小程序中的具体用途
4. 一段最简单的代码示例
5. 常见的相关概念链接

不要用"显然"、"众所周知"这类词。
```

**示例**:把 `【概念】` 填成 `setData` 或 `async/await`,试试看。

---

### 📝 模板 2:代码翻译器

```
我在学小程序,下面是我看到的代码,看不太懂:

```js
(粘贴代码)
```

请帮我:
1. 逐行翻译成人话(每行加注释)
2. 总结这段代码整体在做什么
3. 哪几行是关键逻辑
4. 如果我想改成 XXX 效果,该怎么改
```

---

### 📝 模板 3:页面生成器

```
我要做一个 __________ 小程序,现在要一个 __________ 页面。

需求:
- 布局:__________(比如 顶部搜索 + 列表 + 底部按钮)
- 数据:__________(比如 商品列表,字段有 name/price/image)
- 交互:__________(比如 点击跳转详情)

请给我完整的 4 个文件代码:
- pages/xxx/xxx.wxml
- pages/xxx/xxx.wxss
- pages/xxx/xxx.js
- pages/xxx/xxx.json

用原生小程序语法,代码要有中文注释。
```

```

> 💡 这是最高频的 Prompt,**保存到记事本!**

我的背景:

- 每天 1-2 小时具体任务

- 哪些坑要避开

```

## 四、"学习循环"工作流(每天这样做)

### 🔄 日常学习循环(推荐)

```
早上 / 学习前:
  └─► 问 AI:"今天我要学 XX,给我一份 30 分钟的速成讲解"

学习中:
  ├─► 遇到术语 → 查 02_名词大辞典 or 问 AI
  ├─► 装工具报错 → 用"报错解决器"Prompt
  ├─► 代码看不懂 → 用"代码翻译器"Prompt
  └─► 想加功能 → 用"页面生成器"Prompt

学习后:
  ├─► 让 AI 帮你出 3 题 → 用"知识点验收器"Prompt
  └─► 记录学到的东西(写进自己的笔记)

每周末:
  ├─► 回看本周笔记
  ├─► 做一个本周的实战小案例
  └─► 问 AI:"我这周学了 XX,下周该学什么?"
```

## 五、AI 学习中的 5 个反模式(别这么干)

### ❌ 反模式 1:"伸手党"
问:"帮我做一个微信抢票小程序"
→ AI 不会直接给你"完整项目",会让你自己学过程。
**正确做法**:拆成小问题,一步步问。

### ❌ 反模式 2:只看 AI 的答案,不动手
→ 学的是"看戏",不是"学戏"。
**正确做法**:每个答案后面,**自己跑一遍**。

### ❌ 反模式 3:不验证 AI 的输出
AI 会"幻觉",编造不存在的 API。
**正确做法**:关键代码自己跑,看微信官方文档确认。

### ❌ 反模式 4:问题太模糊
"为什么不行?" —— AI 不是你肚子里的蛔虫。
**正确做法**:贴代码、贴报错、说环境。

### ❌ 反模式 5:碰到难题就换方向
"原生太难了,学 Taro 吧" —— 两周后"Taro 也难,学 Uni-app"。
**正确做法**:卡住 30 分钟 → 问 AI → 还不行 → 发帖求助 → 第二天再战。

## 六、高级用法:让 AI 做你的"陪练"

### 🥋 AI 角色扮演:面试官

```
你现在扮演一位有 5 年经验的小程序面试官。

我在面试初级小程序开发岗位。
请你问我 5 个由浅入深的问题,我回答后你给出评分和改进建议。
问题不要太理论,要贴近实战。

开始第一题。
```

### 🎤 AI 做你的"代码审查员"

```
这是我写的一个小程序页面代码:

```js
(粘贴)
```

请以资深工程师的视角 review:
1. 有没有 bug
2. 有没有更优雅的写法
3. 性能有问题吗(setData 频率)
4. 边界情况是否考虑
5. 打分(1-10)并给出提升建议
```

### 🔍 AI 做你的"阅读加速器"

```
这是我找到的一篇小程序教程,太长了我读不完:
【文章链接或内容】__________

请帮我:
1. 用 5 个要点总结核心内容
2. 告诉我最值得记住的 3 个结论
3. 有没有和我学习目标无关的部分可以跳过
```

### 💼 AI 做你的"产品经理"

```
我想做一个 __________ 小程序,但不知道具体做什么功能。

请扮演资深产品经理,帮我:
1. 问我 5 个问题,了解我的目标用户和场景
2. 基于我的回答,设计 MVP 的核心功能(不超过 5 个)
3. 画出主要页面的线框图描述
4. 给我一个 2 周能做完的精简版方案

现在请开始问第一个问题。
```

## 九、问 AI 的 "3C 原则"

### Context(上下文)
说清楚你的背景、环境、现在在做什么。

### Clarity(清晰)
问题要具体,**一次只问一件事**。

### Constraint(约束)
给出要求(用什么语言、什么风格、多长)。

**示例**:
```
【Context】 我是零基础小白,在学第 3 周,用原生小程序
【Clarity】 我想做一个"点击按钮计数 +1"的效果
【Constraint】 给我完整的 4 个文件代码,带中文注释,不要用 ES6 的解构
```

## 十、学习节奏建议

- **每天 1-2 小时**:比每周一次 8 小时更有效
- **每周至少 1 次实战**:哪怕只花 1 小时
- **每月总结一次**:问 AI "帮我出 10 道本月知识综合题"

## 十一、最后的大白话提醒 💬

> **AI 是"副驾驶",不是"司机"。**
> 你握方向盘 → 它给导航提示
> 你踩油门 → 它提醒你别超速
> **它永远不会代你开到终点。**

真正的学习,**发生在你动手的那一刻**。
