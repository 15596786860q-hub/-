# 闲鱼店铺运营自动化架构

> 面向**技术服务卖家**（卖虚拟服务、定制开发、脚本工具、知识付费）的一套**半自动化运营体系**。
>
> 核心原则：**内容侧全自动 · 客服侧人机协同 · 流量侧合规第一**。
> 不做：批量私信、矩阵养号、改销量、绕风控——这些短期看似能放量，长期都是炸号节奏。

---

## 一、为什么要做"自动化"？先看清现状

很多卖家把"自动化"理解成"用脚本批量发商品/批量打招呼"，这是**最容易被风控的路径**，也是闲鱼明令禁止的。

真正应该自动化的是这四类**重复性高、价值低、决策少**的工作：

| 工作类型 | 每天耗时 | 决策含量 | 适合自动化 |
|---|---|---|---|
| 写主图文案 / 详情页 | 1-2 小时 | 低 | ✅ 强烈推荐 |
| 同款商品多平台同步 | 30-60 分钟 | 低 | ✅ 推荐 |
| 关键词擦亮 / 上下架 | 20-30 分钟 | 低 | ⚠️ 半自动（避开风控） |
| 客服首轮咨询应答 | 1-3 小时 | 中 | ⚠️ 半自动（话术库 + 人工把关） |
| 订单跟进 / 售后提醒 | 30 分钟 | 中 | ✅ 推荐 |
| 数据复盘 / 选题分析 | 1 小时 | 高 | ✅ 推荐 |
| 议价 / 方案沟通 / 报价 | 不定 | 极高 | ❌ 必须人工 |
| 引流到微信 | 不定 | 极高 | ❌ 必须人工（避封号） |

**结论**：能自动化的是"内容生产 + 数据分析 + 流程提醒"，**不能自动化的是"人与人的真实对话"**。

## 三、模块详解

### 模块 ① 选品与素材中台（本地数据库）

**目标**：所有"原料"集中存储，避免每次发商品都从零想标题、找图。

**核心文件结构**：

```
xianyu-ops/
├── sku/
│   ├── sku_001_数据采集.json
│   ├── sku_002_签到脚本.json
│   └── ...
├── keywords/
│   ├── 高频词.csv          # 闲鱼搜索高频词
│   ├── 长尾词.csv          # 转化率高的长尾词
│   └── 违禁词.csv          # 必须规避的敏感词
├── images/
│   ├── 主图模板/           # PSD/Figma 模板
│   ├── 案例截图/           # 真实交付案例
│   └── 二维码/             # 微信引流图（不直接放主图）
└── competitors/
    └── 竞品监控.csv        # 同类卖家价格/标题/销量
```

**SKU JSON 示例**：

```json
{
  "sku_id": "sku_001",
  "category": "数据采集",
  "title_keywords": ["爬虫", "数据抓取", "Excel 导出", "反爬"],
  "price_range": [200, 3000],
  "delivery_days": [1, 7],
  "selling_points": [
    "支持 1 万条以内一次性交付",
    "结构化输出 Excel/JSON",
    "提供采集脚本源码"
  ],
  "use_cases": ["电商比价", "招聘信息聚合", "舆情监控"],
  "case_images": ["images/案例截图/case_001.png"],
  "faq_refs": ["faq_交付周期", "faq_是否提供源码"]
}
```

**自动化程度**：手工录入一次，反复调用。
**工时节省**：发商品时不再从零想文案，**节省 70% 文案时间**。

### 模块 ② AI 内容工厂（核心提效模块）

**目标**：根据 SKU + 关键词，自动批量生成「主图文案 / 标题 / 详情页 / 卖点话术」。

**技术栈**：
- Python 脚本（本地）
- DeepSeek / Kimi / GPT API（按 token 计费，月成本 < ¥50）
- Pillow（主图合成，可选）

**主流程**：

```python
# 伪代码
sku = load_sku("sku_001")
keywords = pick_top_keywords(sku.category, n=5)  # 从词库挑当周热词

# 1. 生成 5 个备选标题
titles = ai.generate_titles(sku, keywords, count=5)

# 2. 生成主图四宫格文案
main_image_text = ai.generate_main_image_copy(sku)

# 3. 生成详情页（分段：痛点 → 服务 → 案例 → 流程 → 价格 → FAQ）
detail = ai.generate_detail_page(sku)

# 4. 输出到待发布目录
save_draft(sku.sku_id, titles, main_image_text, detail)
```

**已有素材可直接复用**：
- 你仓库里的 `闲鱼主图AI提示词_100个.md` 直接作为主图 prompt 库
- 服务清单里的 6 大类目就是天然的 SKU 分组

**自动化程度**：✅ 全自动。运行一次，生成一周的发布草稿。
**工时节省**：每个商品从 30 分钟 → 3 分钟。

### 模块 ③ 上架 SOP（半自动 · 人工执行）

**为什么不全自动？** 闲鱼对"自动化发布工具"是明确禁止的，封号风险极高。所以这一步**只做辅助**，不做替代。

**正确做法**：

| 操作 | 工具 | 频率 |
|---|---|---|
| 草稿展示 | 本地 Web 页面（Flask 起一个本地服务） | 每天 |
| 复制粘贴到闲鱼 App | 手机操作 | 每次发布 |
| 主图上传 | 手动选图 | 每次发布 |
| 擦亮（重新编辑） | 手动，每天 1-2 个 | 每天 |
| 价格微调 | 手动 | 按需 |

**本地草稿页**长这样（伪截图）：

```
┌───────────────────────────────────────────────────┐
│ 待发布草稿  ▽ 数据采集  ▽ 价格 800-3000           │
├───────────────────────────────────────────────────┤
│ [标题1] 网站数据采集 1万条 24h交付 反爬...  [复制]│
│ [标题2] 爬虫定制 Excel导出 全套源码 售...   [复制]│
│ [主图1] 见 generated/sku001_v3.png            [打开]│
│ [详情]  ════════════════════════════════════ [复制]│
│         你是否遇到这样的情况：                    │
│         网站数据想批量获取，但找不到靠谱的人...   │
│         ════════════════════════════════════      │
└───────────────────────────────────────────────────┘
```

**关键纪律**：
- 一天**最多发布 5 个新品**，超过容易触发限流
- 同 SKU 不同标题/主图，间隔 **3 天以上**再发，避免被判同质
- 擦亮选**早 9 点 / 晚 8 点**两个高峰时段

### 模块 ④ 客服话术库（半自动 · 人机协同）

**目标**：90% 的首轮咨询是重复问题，用话术库一键应答；剩下 10% 复杂沟通必须人工。

**话术库结构**（建议存在飞书多维表格 / Notion / 本地 markdown）：

| 场景 | 触发关键词 | 话术 | 备注 |
|---|---|---|---|
| 询价 | "多少钱" "价格" | "您好，这个具体看需求复杂度，方便简单说下…" | 必须人工接管 |
| 周期 | "几天" "多久" | "常规 3-5 天交付，急单可加急…" | 可自动 |
| 是否提供源码 | "源码" "代码" | "提供完整源码 + 部署文档…" | 可自动 |
| 售后 | "售后" "维护" | "1000 元以下 7 天，1000-5000 元 15 天…" | 可自动 |
| 是否做某类 | "能不能做 XXX" | "需要先了解下具体场景，方便加微信沟通吗？" | 引流话术 |
| 砍价 | "便宜点" "能不能少" | "价格已经是独立开发者底价了，但可以…" | 必须人工 |

**实现方式（推荐两种）**：

**方案 A · 轻量级（推荐新手）**：
- iOS：用「快捷指令」+「键盘文本替换」
- Android：用「FlyMe 文本替换」或第三方输入法
- 输入 `..jq` 自动展开成「询价话术」

**方案 B · 进阶**：
- Mac/PC 用 [Espanso](https://espanso.org/)（开源 · 免费 · 跨平台）
- 在闲鱼网页版聊天时直接触发

**绝对不做**：
- ❌ 在闲鱼 App 里跑自动回复机器人
- ❌ 用模拟器/脚本批量回消息
- ❌ 让 AI 直接接管闲鱼对话

**为什么？** 闲鱼通过设备指纹 + 行为特征识别机器人，一旦判定，轻则限流，重则封号。

### 模块 ⑤ 闲鱼平台（人工主战场）

这一层是**人工 + App** 操作，自动化只起辅助作用。

**每日操作清单**（建议打印贴墙上）：

```
☐ 09:00  擦亮 2 个商品（选昨日曝光低的）
☐ 09:30  发布 1 个新品（用昨晚生成的草稿）
☐ 10:00  回复积压咨询（话术库辅助）
☐ 12:00  午休前再回一波咨询
☐ 14:00  处理订单 / 跟进交付
☐ 18:00  擦亮 2 个商品 + 发布 1 个新品
☐ 20:00  晚高峰回咨询
☐ 22:00  数据复盘（看哪些商品曝光高/转化低）
```

**红线清单（一条都不能碰）**：
- 不发联系方式（微信号、QQ、电话）在主图/详情/聊天首条
- 不在主图放二维码
- 不批量复制同一段话发给不同买家
- 不在 24 小时内修改价格超过 3 次
- 不用同一台设备登录多个账号（**这条最容易炸号**）

### 模块 ⑥ 微信沉淀池（核心商业资产）

**为什么必须沉淀到微信？**
- 闲鱼账号是平台的，微信好友是你的
- 闲鱼无法做长期客户运营，微信可以
- 复购、转介绍都发生在微信

**合规引流路径**（**不要想着"自动化"这一步，每一次引流都必须人工判断**）：

```
买家咨询  →  你判断意向度
              │
              ├─ 低意向 → 平台内话术应答即可
              │
              └─ 高意向 → "这个需求比较复杂，方便加微信详细沟通吗？我把案例发您看下"
                            │
                            └─ 同意 → 私信发微信号（注意：发文字，不发二维码图）
                                       │
                                       └─ 加上后第一句："您好我是闲鱼上的XX"（建立关联）
```

**沉淀后的微信侧自动化**（这部分可以适度自动化）：

| 任务 | 工具 | 自动化程度 |
|---|---|---|
| 新好友自动通过 | 微信本身设置 | ✅ |
| 标签分组 | 手动打标 | ⚠️ 必须人工 |
| 朋友圈发案例 | 微信本身 + 素材库 | 半自动 |
| 周期性触达老客户 | 飞书多维表格 + 提醒 | 半自动 |
| 群发问候（节日） | **不推荐**，违反微信规则 | ❌ |

**老客户复购提醒表**（飞书表格）：

| 客户 | 上次成交日 | 项目 | 90天后提醒 | 是否触达 |
|---|---|---|---|---|
| 王某 | 2026-02-15 | 数据采集 | 2026-05-16 | ☐ |
| 李某 | 2026-03-01 | 签到脚本 | 2026-05-30 | ☐ |

设置飞书自动提醒，到日期推送到企微/邮箱。

### 模块 ⑦ 数据复盘看板（决定下周做什么）

**目标**：每周用 30 分钟，看清"哪些品在赚钱，哪些品在拖后腿"。

**采集哪些数据**（闲鱼"我的"页面 → 商品管理 → 数据中心 手动导出/截图）：

| 指标 | 来源 | 频率 |
|---|---|---|
| 曝光量 | 闲鱼数据中心 | 每日 |
| 点击量 | 闲鱼数据中心 | 每日 |
| 咨询量 | 自己记录 | 每日 |
| 成交量 / GMV | 闲鱼订单 | 每日 |
| 客单价 | 计算 | 每周 |
| 引流到微信数 | 自己记录 | 每周 |

**本地分析脚本**（Python + pandas + matplotlib）：

```python
# 每周日 22:00 自动跑一次
weekly_data = load_csv("data/2026-W20.csv")

# 1. 算每个商品的转化漏斗
funnel = calc_funnel(weekly_data)
# 曝光 → 点击 (CTR) → 咨询 (CMR) → 成交 (CVR)

# 2. 找 Top 5 / Bottom 5
top_5 = funnel.nlargest(5, "GMV")
bottom_5 = funnel.nsmallest(5, "CTR")

# 3. 自动生成周报 PDF（复用现有的 _build_pdf.py 思路）
generate_weekly_report(top_5, bottom_5, output="reports/2026-W20.pdf")
```

## 六、ROI 测算（保守估算）

**自动化前**（每天 8 小时）：
- 写文案：2h
- 发商品/擦亮：1h
- 回咨询：3h
- 订单跟进：1h
- 数据复盘：1h
- **有效产出时间：很少**

**自动化后**（每天 6 小时）：
- AI 生成草稿审核：30min（节省 90 分钟）
- 发商品/擦亮：1h
- 回咨询（话术库）：1.5h（节省 90 分钟）
- 订单跟进：1h
- 数据复盘（自动周报）：10min（节省 50 分钟）
- **多出 2-3 小时** → 用来接单干活/学新技术

**收益层面**：
- 同样时间能维护 SKU 数量从 20 → 60+
- 咨询响应速度从平均 1 小时 → 15 分钟内（转化率 +30%）
- 老客户复购率（有提醒触达） → +50%

**保守估计：月 GMV 提升 50%-100%**。

## 七、风险与红线（必读）

### 闲鱼平台规则红线
1. **不批量发布**：脚本批量上架 = 直接封号
2. **不矩阵养号**：一台设备多账号 = 直接封号
3. **不站外引流写在明处**：主图/详情写"加微信" = 限流 + 商品下架
4. **不刷单不改销量**：一旦查到 = 永封 + 资金冻结
5. **不发违禁品/服务**：你服务清单里"我不接"那 7 条，一条都不能碰

### 微信侧红线
1. 不用 ItChat / WeChaty 等老接口（早就不安全了）
2. 不用 PC 端 hook 工具批量加好友
3. 不群发营销消息（一封一个准）

### 数据安全
1. 客户数据本地存储，不上传第三方云
2. 微信号、订单截图等敏感信息不进 git 仓库
3. AI API key 用 `.env` 文件，加入 `.gitignore`

## 八、最终建议
