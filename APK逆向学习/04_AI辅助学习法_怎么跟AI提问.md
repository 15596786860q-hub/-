# 04 | AI 辅助学习法:怎么跟 AI 高效对话

> **这是最重要的一篇文档。**
> 学会用 AI,你的学习速度能 **快 10 倍**。学不会用 AI,有神仙老师也没用。

---

## 一、AI 能给你什么?不能给你什么?

### ✅ AI 擅长做的
- 📖 **解释任何概念**(大白话、带类比)
- 🔧 **生成代码/脚本**(Frida Hook、Python 解密脚本)
- 🐛 **调试报错**(粘贴错误信息,90% 能解决)
- 🧠 **分析代码逻辑**(粘贴一段 smali/Java,让它翻译)
- 📝 **总结知识点**(学完一块让它帮你总结)
- 🎯 **出练习题**(让它给你出题验证学习效果)

### ❌ AI 不擅长做的
- 🙅 **替你操作**(安装工具、连手机、点击界面 —— 必须你自己动手)
- 🙅 **保证绝对正确**(可能有"幻觉",关键结果自己验证)
- 🙅 **代替思考**(你不动脑,就是"喂答案",学不会)

---

## 二、核心心法 ⭐⭐⭐

### 🎯 心法 1:**学 → 做 → 问 → 再做**

```
      ┌─ 学(看文档 / 让 AI 讲)
      │
      ▼
    做一下(动手试)
      │
      ▼
   遇到问题 ──→ 问 AI
      │              │
      │    ←─────────┘
      ▼
   再做(用 AI 给的思路)
      │
      ▼
    学会了 ✓
```

**最致命的错误**:看了就算学了。**必须动手**。

### 🎯 心法 2:**问具体的,不问抽象的**

❌ 烂问题:"怎么学逆向?"(太宽,AI 只能给泛泛的答案)
✅ 好问题:"我装好了 jadx,想分析一个去广告的 App,具体怎么找到广告相关的代码?"

### 🎯 心法 3:**给 AI 足够的上下文**

AI 不知道你的:
- 操作系统(Win? Mac?)
- 手机型号(有没有 Root?)
- 现在学到哪一步
- 之前做过什么

**每次问,先交代清楚。**

### 🎯 心法 4:**AI 答不对,换种问法**

如果 AI 给的答案不对/听不懂,可以说:
- "我是零基础小白,能用更简单的话解释吗?"
- "能给个生活类比吗?"
- "这个解释里 XX 我不懂,单独展开讲"

## 三、7 个神级 Prompt 模板(复制即用)

### 📝 模板 1:概念解释器

```
你是一位资深的安卓逆向工程师,现在我是零基础小白。

请用大白话解释下面这个概念:
【概念】:__________

要求:
1. 一句话核心定义
2. 一个生活化类比
3. 在 APK 逆向中的具体用途
4. 举一个最简单的例子
5. 常见的相关术语链接

不要用"显然"、"众所周知"这类词。
```

**示例**:把 `【概念】` 填成 `JNI`,试试看。

---

### 📝 模板 2:代码翻译器

```
我在逆向一个 Android App,下面是 jadx 反编译的代码:

【代码】:
```java
(粘贴代码)
```

请帮我:
1. 逐行翻译成人话(每行加注释)
2. 总结这段代码整体在做什么
3. 哪几行是关键逻辑(比如加密、校验、网络请求)
4. 如果我想 Hook 它,应该 Hook 哪个方法、怎么做
```

---

### 📝 模板 3:Frida 脚本生成器

```
我要 Hook 一个 App,需求如下:

- App 包名:__________
- 目标类名:__________
- 目标方法:__________
- 目的(想看参数 / 改返回值 / 调用它):__________

请帮我:
1. 写一份 Frida Hook 脚本(附中文注释)
2. 告诉我怎么运行它(完整命令)
3. 可能遇到的报错和解决方案
```

---

### 📝 模板 4:报错解决器

```
我在做 __________(简述正在做什么)。

我运行的命令:
```
(命令)
```

报错信息:
```
(完整错误)
```

我的环境:
- 操作系统:__________
- 工具版本:__________
- 目标 App / 设备:__________

我已经尝试过:
- __________
- __________

请帮我分析原因 + 给出 2-3 个解决方案。
```

> 💡 这是最高频的 Prompt,**保存到记事本!**

---

### 📝 模板 5:学习路径规划器

```
我是零基础,学 APK 逆向。

我的背景:
- 会的编程语言:__________(或零基础)
- 可用时间:每天 __ 小时
- 目标(几个月内):__________(比如能给某 App 写爬虫)

请帮我设计一个未来 4 周的学习计划,要求:
- 每周目标
- 每天 1-2 小时具体任务
- 每周末的验收标准
- 用什么工具
- 哪些坑要避开
```

---

### 📝 模板 6:知识点验收器

```
我刚学完 __________(主题)。

请用以下方式检测我是否真的学会:
1. 出 3 个概念题(让我用自己的话解释)
2. 出 2 个情境题(给我个场景让我说怎么处理)
3. 出 1 个动手题(一个小任务让我实操)
4. 每题后告诉我正确答案和扣分点
```

---

### 📝 模板 7:案例分析器

```
我想学习一个经典的逆向案例,请帮我设计:

场景:__________(比如"去某 App 开屏广告")

请按以下结构输出:
1. 这个案例的**难度**(1-5 星)
2. **需要的前置知识**(列出来)
3. **用到的工具**(列出来)
4. **详细步骤**(每步都告诉我"做什么 + 为什么 + 预期结果")
5. **可能遇到的问题**
6. **做完后能学到什么**

注意:我是小白,步骤要细到每一次点击。
```

## 四、"学习循环"工作流(每天这样做)

### 🔄 日常学习循环(推荐)

```
早上 / 学习前:
  └─► 问 AI:"今天我要学 XX,给我一份 30 分钟的速成讲解"

学习中:
  ├─► 遇到术语 → 查 02_名词大辞典 or 问 AI
  ├─► 装工具报错 → 用"报错解决器"Prompt
  └─► 代码看不懂 → 用"代码翻译器"Prompt

学习后:
  ├─► 让 AI 帮你出 3 题 → 用"知识点验收器"Prompt
  └─► 记录学到的东西(写进自己的笔记)

每周末:
  ├─► 回看本周笔记
  ├─► 做一个本周的实战小案例
  └─► 问 AI:"我这周学了 XX,下周该学什么?"
```

## 五、AI 学习中的 5 个反模式(别这么干)

### ❌ 反模式 1:"伸手党"
问:"帮我破解这个 App 的 VIP"
→ AI 不会直接给你"破解成品",会让你自己学过程。
**正确做法**:拆成小问题,一步步问。

### ❌ 反模式 2:只看 AI 的答案,不动手
→ 学的是"看戏",不是"学戏"。
**正确做法**:每个答案后面,**自己跑一遍**。

### ❌ 反模式 3:不验证 AI 的输出
AI 会"幻觉",编造不存在的函数、错误的 API。
**正确做法**:关键代码自己跑一跑,报错就再问。

### ❌ 反模式 4:问题太模糊
"这个为什么不行?" —— AI 不是你肚子里的蛔虫。
**正确做法**:贴命令、贴报错、说环境。

### ❌ 反模式 5:碰到难题就换方向
"Frida 太难了,我不学了学抓包" —— 两周后又说"抓包太难不学了"。
**正确做法**:卡住 30 分钟 → 问 AI → 还不行 → 发帖求助 → 第二天再战。

## 六、高级用法:让 AI 做你的"陪练"

### 🥋 AI 角色扮演

```
你现在扮演一位有 10 年经验的安卓逆向面试官。

我在面试初级逆向工程师岗位。
请你问我 5 个由浅入深的问题,我回答后你给出评分和改进建议。
问题不要太理论,要贴近实战。

开始第一题。
```

### 🎤 AI 做你的"代码审查员"

```
这是我写的一个 Frida Hook 脚本,请以资深工程师的视角 review:
```javascript
(你的脚本)
```

请指出:
1. 有没有 bug
2. 有没有更优雅的写法
3. 边界情况是否考虑
4. 打分(1-10)并给出提升建议
```

### 🔍 AI 做你的"阅读加速器"

```
这是我找到的一篇逆向教程,太长了我读不完:
【文章链接或内容】__________

请帮我:
1. 用 5 个要点总结核心内容
2. 告诉我最值得记住的 3 个结论
3. 有没有和我学习目标无关的部分可以跳过
```

## 十、最后的大白话提醒 💬

> **AI 是"副驾驶",不是"司机"。**
> 你握方向盘 → 它给导航提示
> 你踩油门 → 它提醒你别超速

真正的学习,**发生在你动手的那一刻**。
