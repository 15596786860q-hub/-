## 🎯 案例 1:Hello CrackMe ⭐(入门第一枪)

### 目标
破解一个故意给你练习的小 App(CrackMe),输入密码就能过关。

### 需要的技能
- jadx 基本使用
- 会读 Java 代码

### 步骤
1. 去 GitHub 搜 "android crackme" 下载一个(推荐 OWASP 的 UnCrackable-Level1)
2. 用 jadx 打开
3. 找到密码校验逻辑(搜"密码错误"之类的字符串)
4. 看明文密码,或找到校验函数逻辑后手算

### 能学到
- 完整跑通一次"拿 APK → 反编译 → 找逻辑"
- 建立逆向的"肌肉记忆"

### AI 辅助 Prompt
> "我下载了 UnCrackable-Level1.apk,用 jadx 打开后看到很多类,帮我分析哪个是主入口?"

## 🎯 案例 2:去掉小 App 的开屏广告 ⭐⭐

### 目标
找一个有开屏广告的小众 App,把开屏广告去掉。

### 需要的技能
- jadx + apktool
- Smali 修改
- 重打包 + 签名

### 步骤
1. 选目标(**避开大厂**,找 3 天内能看完代码的小 App)
2. jadx 打开,搜 "splash" / "ad" / "广告"
3. 定位广告显示的入口
4. apktool 解包
5. 修改 smali:让广告方法直接 return,或改跳转逻辑
6. 重打包 + apksigner 签名
7. 装机验证

### 常见坑
- 签名不匹配导致装不上 → 先卸载原版
- 改 smali 时改错 `.locals` 寄存器数量 → AI 帮你检查

### AI 辅助 Prompt
> "这段 smali 代码是广告的入口,我想让它直接返回,怎么改?注意寄存器数量"

## 🎯 案例 3:修改 App 名称 + 图标 ⭐

### 目标
把一个 App 的名字改成你的名字,图标换成你喜欢的。

### 需要的技能
- apktool 解包
- XML 编辑

### 步骤
1. apktool 解包
2. 修改 `res/values/strings.xml` 的 `app_name`
3. 替换 `res/mipmap-*/ic_launcher.png`
4. 重打包

### 能学到
- 资源文件结构
- 基础改包流程

### 难度
最简单的,作为**练手热身**。

## 🎯 案例 4:Frida Hello World ⭐⭐

### 目标
写出你人生第一个 Frida Hook 脚本,Hook 一个方法并打印参数。

### 需要的技能
- Frida 环境搭建
- JavaScript 基础(不用深,会改参数就行)

### 步骤
1. 电脑装 frida-tools,手机跑 frida-server
2. 随便找一个 App(或自己写一个 Demo)
3. 用 jadx 找一个方法名
4. 写 Hook 脚本打印参数 / 返回值
5. `frida -U -f 包名 -l hook.js`

### 能学到
- Frida 基础工作流
- Hook 的威力

### AI 辅助 Prompt
> "帮我写一个最简单的 Frida Hook 脚本,Hook com.test.app 的 Login 类的 check 方法,打印所有参数"

## 🎯 案例 5:Hook 某 App 的登录接口抓密码 ⭐⭐⭐

### 目标
找一个**你自己账号**的 App,Hook 登录按钮,抓到明文密码(仅个人学习,不能对他人做!)

### 需要的技能
- Frida Hook Java 层
- 读 Java 代码能力

### 步骤
1. jadx 打开 App,找登录 Activity
2. 定位登录按钮点击事件的代码
3. 追到 `login(username, password)` 这种方法
4. Frida Hook 这个方法,打印参数

### 能学到
- **从静态到动态**的完整分析链路
- 这是**写爬虫**的基础技能

### 重要提醒 ⚠️
只能 Hook 自己的账号,不能对别人做。

## 🎯 案例 6:抓 HTTPS + 绕过 SSL Pinning ⭐⭐⭐

### 目标
找一个大厂 App(比如一个你不介意被研究的小 App),抓到它的 API 请求。

### 需要的技能
- Charles / mitmproxy
- Frida Universal Pinning Bypass

### 步骤
1. Charles 配好,装证书到系统
2. 打开 App,发现抓不到 → 有 Pinning
3. 用 Frida 跑通用绕过脚本
4. 成功抓到请求
5. 分析参数:哪些是明文,哪些是签名

### 能学到
- HTTPS 抓包全流程
- 真实世界的 App 如何保护接口

### AI 辅助 Prompt
> "我用 Frida 跑了 frida-universal-pinning-bypass 但还是抓不到,App 会退出,怎么办?"

## 🎯 案例 7:还原一个简单的签名算法 ⭐⭐⭐⭐

### 目标
分析一个 App 的 API 签名(sign 参数)是怎么生成的,用 Python 复现。

### 需要的技能
- 静态 + 动态分析
- Python + 加密库
- 耐心(这是最考验耐心的一步)

### 步骤
1. 抓包拿到一个请求,看 `sign` 字段
2. jadx 找签名生成的代码(搜 "sign"、"signature")
3. Frida Hook 签名函数,对比输入和输出
4. 分析算法(通常是 MD5(参数排序 + key))
5. 用 Python 实现
6. 测试:你生成的 sign 和 App 生成的一致

### 能学到
- **爬虫工程师的核心技能**
- 从源码到复现的完整链路

### 难点
- 参数排序规则
- 盐值(salt)藏在哪里

## 🎯 案例 8:分析 Native 层(so)的加密算法 ⭐⭐⭐⭐

### 目标
App 把关键算法藏在 `.so` 文件里,你要分析出来。

### 需要的技能
- Ghidra / IDA
- ARM 汇编基础
- JNI 机制

### 步骤
1. jadx 发现 `native` 方法 → 说明函数在 so 里
2. `lib/arm64-v8a/libXXX.so` 用 Ghidra 打开
3. 根据函数名(静态注册)或 `RegisterNatives`(动态注册)找到目标
4. F5 反编译看 C 代码
5. Frida Hook 验证分析结果

### 能学到
- Native 层分析流程
- 这是**高级爬虫**的必备

## 🎯 案例 9:脱壳一个加固 App ⭐⭐⭐⭐⭐

### 目标
找一个用 360 / 梆梆 / 爱加密加固的 App,把真实代码脱出来。

### 需要的技能
- 脱壳工具(FART / BlackDex)
- Root 环境
- 耐心

### 步骤
1. 查壳工具识别加固类型
2. 选合适的脱壳方案
3. 跑脱壳机,dump 出 DEX
4. 修复 DEX(某些壳脱出来是"残缺"的)
5. jadx 打开 dump 出的 DEX

### 难点
- 脱壳机自己也要折腾半天环境
- 不同加固要用不同方案

### 能学到
- 加固原理
- 对抗思维

## 🎯 案例 10:Unidbg 调 so 做爬虫 ⭐⭐⭐⭐⭐

### 目标
不装手机、不开模拟器,直接在服务器上用 Unidbg 调用 App 的签名 so,批量采集数据。

### 需要的技能
- 所有前面技能的综合
- Java(用 IDEA)

### 步骤
1. 从 App 里抽出目标 so
2. Unidbg 项目中新建一个类加载 so
3. 疯狂补环境(会报缺这个缺那个)
4. 调用成功,跑通一次签名
5. 封装成 HTTP 接口(比如用 SpringBoot)
6. Python 爬虫请求这个接口拿签名

### 能学到
- 这是逆向工程师**最值钱的技能**之一
- 一个完整 Unidbg 项目能在爬虫圈卖几万

## 💡 实战通用心法

### 1. **从小 App 开始,别碰大厂**
微信、抖音、淘宝 —— 三个月也搞不定,别浪费时间。

### 2. **同一个案例做 3 遍**
第一遍做完,过一周再做一遍(不看笔记),第三遍写一篇教程给别人。

### 3. **记录所有报错**
建一个"错误库",以后遇到能秒查。

### 4. **别怕做错**
改坏 App 了大不了重装,改错 smali 大不了重来 —— **成本几乎为零**。

### 5. **加入社区**
看雪 / 吾爱破解发帖,遇到问题有人帮。
