# 01 | 知识架构:一张图看懂全貌

> 看完这篇,你就能在脑子里有一张"地图",知道自己每天在学哪一块。

---

## 一、最简模型:APK 逆向 = 拆快递 📦

```
    [APK 文件]  ← 一个密封的快递箱
        │
        │ 用工具拆开(apktool / jadx)
        ▼
    [里面的东西]
    ├── 说明书(AndroidManifest.xml)
    ├── 主要货物(classes.dex = 代码)
    ├── 附件(resources / assets = 图片文字)
    └── 底层零件(lib/*.so = C/C++ 编译的库)
        │
        │ 看货物怎么工作(静态阅读 + 动态跟踪)
        ▼
    [搞懂 App 的逻辑]
        │
        │ 修改/Hook/复现算法
        ▼
    [达到你的目的:去广告 / 抓包 / 还原签名]
```

## 二、金字塔分层:从小白到高手的 5 层 🏔

```
                       ┌─────────────────────────┐
                       │   第 5 层  加固对抗      │  ← 大神级
                       │  (脱壳 / 反反调试 / VMP) │
                       ├─────────────────────────┤
                       │   第 4 层  Native 逆向    │  ← 进阶
                       │   (so 库 / C++ / ARM)   │
                       ├─────────────────────────┤
                       │   第 3 层  动态分析       │  ← 核心
                       │   (Frida Hook / 抓包)   │
                       ├─────────────────────────┤
                       │   第 2 层  静态分析       │  ← 入门
                       │   (jadx / apktool)     │
                       ├─────────────────────────┤
                       │   第 1 层  基础知识       │  ← 地基
                       │   (Android + 编程)      │
                       └─────────────────────────┘
```

### 每一层是什么?一句话解释

| 层级 | 名字 | 一句话 | 对应目标 |
|------|------|-------|---------|
| **1** | 基础知识 | 学 Android 怎么运行、学看懂代码 | 能读懂 Java |
| **2** | 静态分析 | 把 APK 拆开,**不运行**直接读它的代码 | 去广告、破弹窗 |
| **3** | 动态分析 | 让 App **跑起来**,在运行时改它的行为 | 抓包、爬虫、绕过登录 |
| **4** | Native 逆向 | 分析 C/C++ 写的 `.so` 库(更底层) | 还原加密算法 |
| **5** | 加固对抗 | App 被加固保护了,你得"脱壳"才能看 | 分析商业级 App |

## 三、详细技能树 🌳

```
APK 逆向
│

│   ├── APK 解包工具
│   │   ├── jadx-gui       ← 一键反编译,最常用
│   │   ├── apktool        ← 能重打包
│   │   └── dex2jar        ← 老牌工具
│   ├── 代码审计
│   │   ├── 看 AndroidManifest(找入口/权限)
│   │   ├── 搜字符串(找关键逻辑)
│   │   ├── 追踪函数调用链
│   │   └── 识别加密算法
│   └── 重打包
│       ├── 改 Smali 代码
│       ├── 重新签名(apksigner)
│       └── 安装测试
│
├── 🎣 ③ 动态分析层(跑起来抓)
│   ├── Hook 框架(最重要!)
│   │   ├── Frida ⭐⭐⭐    ← 神器,必学
│   │   ├── Xposed/LSPosed ← 需要 Root
│   │   └── Objection      ← Frida 的"傻瓜版"
│   ├── 抓包工具
│   │   ├── Charles / Fiddler    ← 图形化,抓 HTTP
│   │   ├── mitmproxy            ← 命令行,可编程
│   │   └── Wireshark            ← 抓底层 TCP/UDP
│   ├── 证书处理
│   │   ├── 安装用户证书 / 系统证书
│   │   └── 绕过 SSL Pinning(证书锁定)
│   └── 调试
│       ├── adb logcat 看日志
│       └── smalidea 断点调试
│
├── 🔧 ④ Native 层(更底层)
│   ├── JNI 机制
│   │   ├── Java 怎么调用 C/C++ 代码
│   │   └── 静态注册 vs 动态注册
│   ├── so 分析
│   │   ├── IDA Pro        ← 收费但最强
│   │   ├── Ghidra         ← 免费开源,NSA 出品
│   │   └── Binary Ninja   ← 界面友好
│   ├── 汇编基础
│   │   ├── ARM / ARM64 指令集
│   │   └── 寄存器、栈、调用约定
│   └── 模拟执行
│       └── Unidbg         ← 脱机跑 so(爬虫党最爱)

└── 🛡 ⑤ 加固对抗层(高级玩家)
    ├── 识别加固类型
    │   └── 360 / 梆梆 / 爱加密 / 腾讯乐固 / 顶象

    │   ├── BlackDex       ← 易用

    ├── 反反调试
    │   ├── 绕过 ptrace 检测

    │   └── 绕过 Root 检测

        ├── VMP 虚拟化保护
        └── 自定义指令集还原
```

## 四、三种主流"套路"(别人都这么干)

### 套路 A:纯静态流派 "看派" 📖
**步骤**:jadx 打开 → 搜关键字 → 读代码 → 还原逻辑
**适合**:简单 App、没加固
**缺点**:遇到混淆就废

### 套路 B:动态 Hook 流派 "钩派" 🎣 ⭐ 主流
**步骤**:跑 App → Frida 注入 → Hook 目标函数 → 拿到明文/密钥
**适合**:90% 的日常场景
**优点**:不管代码多复杂,Hook 一下就出结果

### 套路 C:模拟执行流派 "仿派" 🤖
**步骤**:抽出 so → Unidbg 搭环境 → Python 直接调用
**适合**:批量跑签名(爬虫场景)
**优点**:脱机、快

> 💡 **学习顺序建议**:先学 A(建立阅读能力),再学 B(90% 场景能搞定),进阶学 C。

## 六、每一层对应的"毕业标志" 🎓

| 层级 | 学完的标志(能做到这些才算过关)|
|------|-------------------------------|
| **① 基础** | 能读懂 10 行 Java 代码,能用 adb 连接手机 |
| **② 静态** | 能用 jadx 打开一个 App,搜字符串定位到代码 |
| **③ 动态** | 能用 Frida Hook 一个函数,打印它的参数和返回值 |
| **④ Native** | 能用 IDA 打开 .so 文件,看懂一个简单函数 |
| **⑤ 对抗** | 能对一个 360 加固的 App 脱壳出原始 DEX |
