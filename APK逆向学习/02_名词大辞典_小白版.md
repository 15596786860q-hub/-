## A

### ADB (Android Debug Bridge)
**一句话**:电脑和安卓手机"对话"的工具。
**类比**:像是手机的"遥控器",你在电脑上敲命令,手机上执行。
**常用命令**:
- `adb devices` 看连了哪些设备
- `adb install app.apk` 装 App
- `adb shell` 进入手机的命令行
- `adb logcat` 看 App 的运行日志

### AES
**一句话**:一种**对称加密算法**(加密和解密用同一把钥匙)。
**类比**:你和朋友约好同一个暗号,用它加密信件,也用它解开。
**逆向场景**:很多 App 用 AES 加密数据传输,你要找到"钥匙(Key)"就能解密。

### Android Studio
**一句话**:Google 官方的 Android 开发工具(IDE)。
**逆向中的用途**:主要用来**装模拟器**、调试 smali 代码。

### APK (Android Package Kit)
**一句话**:安卓 App 的安装包文件。
**本质**:一个压缩包(zip),改后缀名 `.apk → .zip` 就能解压看内容。
**包含**:代码、图片、配置文件、底层库,全在里面。

### apktool
**一句话**:把 APK 拆开再装回去的瑞士军刀。
**能做什么**:
- 解包 APK(变成 smali 代码 + 资源文件)
- 修改后重新打包(重打包)
**局限**:只能看 smali(不是 Java),不如 jadx 直观。

### ARM / ARM64
**一句话**:手机 CPU 的"指令集"(就像 x86 是电脑 CPU 的指令集)。
**为什么重要**:逆向 `.so` 文件时,看到的汇编代码就是 ARM 指令。
**ARM** = 32 位,**ARM64** = 64 位(现在新手机几乎都是 ARM64)。

### ARSC (resources.arsc)
**一句话**:APK 里存储资源(图片、字符串)ID 的索引表。

### ART (Android Runtime)
**一句话**:Android 系统运行 App 的"引擎"(安卓 5.0 之后取代 Dalvik)。
**对比**:像 Java 的 JVM(虚拟机),但专门跑 Android 的 DEX。

## B

### Binary Ninja
**一句话**:一款商业的二进制分析工具,界面好看。对标 IDA Pro。

### BlackDex
**一句话**:一款**脱壳工具**,帮你把加固过的 App 还原成原始 DEX。

---

## C

### C / C++
**一句话**:更底层的编程语言,Android 里一般用来写 `.so` 库(性能敏感、或者藏加密算法)。

### CFG (Control Flow Graph)
**一句话**:控制流图。程序里各个代码块怎么跳转的图。
**哪里见**:IDA / Ghidra 打开函数会看到一张流程图,就是 CFG。

### Charles / Fiddler
**一句话**:抓包神器。能看到 App 和服务器之间在传什么数据。
**类比**:像是"邮件检查员",App 发出去的每一封信都被你拆开看了。

### ClassLoader
**一句话**:Java/Android 用来"加载类"的东西。
**为什么重要**:加固 App 常常动态加载代码,要 Hook ClassLoader 才能看到真实代码。

### CrackMe
**一句话**:故意给你练习破解的小程序。
**类比**:像是"锁匠练习用的锁",可以随便撬。

### CTF (Capture The Flag)
**一句话**:网络安全比赛,有一类题就是逆向 APK 找 flag。
**收益**:练技术 + 拿奖金 + 简历加分。

---

## D

### Dalvik
**一句话**:Android 5.0 之前的运行引擎(现在被 ART 取代了,但"Dalvik 字节码"这个词还在用)。

### DEX (Dalvik Executable)
**一句话**:APK 里的**代码文件**,类似电脑上的 `.exe`。
**所有 Java/Kotlin 代码最终都被编译成 DEX**。
**一个 APK 可能有多个 dex**:`classes.dex`、`classes2.dex`……

### dex2jar
**一句话**:把 `.dex` 转成 `.jar`(方便用 Java 工具读)。
**现状**:被 jadx 取代了,但老教程还常提到。

### Dump(转储)
**一句话**:把内存里的数据"扒"出来存成文件。
**场景**:App 把 DEX 藏在内存里,你把它 dump 出来就能看代码了。

---

## E

### ELF
**一句话**:Linux/Android 下的可执行文件格式。`.so` 就是 ELF 格式。

---

## F

### FART
**一句话**:一款**脱壳机**,主动调用每个方法把代码 dump 出来。**脱壳界的老大**。

### Frida ⭐⭐⭐
**一句话**:动态 Hook 神器,**逆向界的瑞士军刀**。
**能做什么**:
- App 正在运行时,你能"插入"代码进去
- 打印任意函数的参数、修改返回值
- 绕过 SSL Pinning、绕过各种检测
**为什么要学**:90% 的现代逆向场景都靠它。
**语言**:用 JavaScript 写 Hook 脚本。

### Flag
**一句话**:CTF 比赛里藏在题目中的"答案字符串",找到它就得分。

---

## G

### Ghidra
**一句话**:**免费**的逆向神器,NSA(美国国家安全局)开源的。
**对标**:IDA Pro(收费几万块),Ghidra 完全免费。

### Gradle
**一句话**:Android 开发的构建工具(开发者用,逆向基本不用管)。

---

## H

### Hash(哈希)
**一句话**:把任意数据"压缩"成固定长度的字符串,**不可逆**。
**常见**:MD5(32 位)、SHA1(40 位)、SHA256(64 位)。
**逆向场景**:登录密码一般都是 hash 后传输的。

### Hook(钩子)
**一句话**:在程序运行时"钩住"某个函数,让你能改它的行为。
**类比**:邮差本来要把信送到 A,你在半路拦截、看信、改写、再送走。
**工具**:Frida、Xposed。

### HTTPS / SSL / TLS
**一句话**:加密的 HTTP。App 和服务器通信大多用 HTTPS。
**抓包难点**:HTTPS 被加密了,要装"证书"才能抓到明文。

---

## I

### IDA Pro
**一句话**:逆向界的"倚天剑",最强的静态分析工具(**收费,几万块**)。
**能做**:把二进制(.so / .exe)反编译成 C 伪代码。
**F5 键**:按 F5 把汇编变成 C 代码,这一招无人不知。

### IDE (Integrated Development Environment)
**一句话**:集成开发环境,就是个写代码的软件(如 Android Studio、VS Code)。

---

## J

### Jadx / jadx-gui ⭐⭐⭐
**一句话**:**最常用的静态分析工具**,把 APK 直接变回 Java 代码。
**为什么好用**:
- 免费开源
- 直接拖入 APK,自动反编译
- 能搜字符串、看类、追函数调用
**小白必装**。

### Java
**一句话**:Android 开发的主流语言之一。
**逆向中**:你必须能**读懂** Java 代码(不必会写)。

### jarsigner / apksigner
**一句话**:给 APK 签名的工具。改完 APK 要重新签名才能装。

### JEB Pro
**一句话**:商业的 Android 逆向工具,smali 分析能力顶级。**收费**。

### JNI (Java Native Interface)
**一句话**:Java 和 C/C++ 之间的"翻译官"。
**为什么存在**:Java 想调用 C 写的代码(通常是为了性能或安全),通过 JNI 实现。
**逆向场景**:很多 App 把加密算法写在 C 里(so 文件),通过 JNI 暴露给 Java 调用。

---

## K

### Kotlin
**一句话**:一门现代语言,Google 2017 年后力推,新 App 基本都用 Kotlin 写。
**逆向中**:Kotlin 编译后还是 DEX,用 jadx 打开能看到(有些语法稍奇怪)。

### Key(密钥)
**一句话**:加密算法用的"钥匙"。

---

## L

### LSPosed
**一句话**:**Xposed 的现代版**,基于 Riru / Zygisk,需要 Root。
**能做**:持久化 Hook(开机就自动注入模块)。

### Linux
**一句话**:Android 的底层就是 Linux 内核。所以 Linux 命令在逆向中常用到。

---

## M

### Magisk
**一句话**:给手机 Root 的神器,且能"隐藏 Root"让 App 检测不到。
**逆向前置**:想玩动态 Hook,手机最好 Root,Root 就用 Magisk。

### Manifest (AndroidManifest.xml)
**一句话**:APK 的"说明书",告诉系统这个 App 有哪些页面、要哪些权限、谁是入口。
**逆向第一步**:打开 APK 先看 Manifest,找到主 Activity(入口)。

### MD5
**一句话**:一种 hash 算法,输出 32 位字符串。
**现状**:已被破解,不安全,但 App 里还很常见。

### mitmproxy
**一句话**:命令行抓包工具,可以用 Python 脚本自动化处理请求。

### 模拟器
**一句话**:电脑上假装自己是安卓手机的软件。
**常用**:Android Studio 自带模拟器、雷电、MuMu、夜神、Genymotion。
**坑**:很多 App 会检测模拟器,你学到后面要学"反模拟器检测"。

---

## N

### NDK (Native Development Kit)
**一句话**:Google 给开发者写 C/C++ 代码用的工具包。你学逆向不会直接用,但要知道 so 是 NDK 编译的。

### Native 代码
**一句话**:C/C++ 写的代码,编译后变成 `.so` 文件。
**对比**:Java/Kotlin → DEX,C/C++ → so。

---

## O

### Objection
**一句话**:基于 Frida 的**自动化工具**,一键执行常见 Hook。
**类比**:Frida 是菜刀,Objection 是带菜刀的"料理机器人",很多事它自动干。

### OLLVM
**一句话**:一种代码混淆工具,让你看不懂代码。
**特征**:控制流被搞得像面条,函数变得乱七八糟。
**高阶话题**:有专门"反 OLLVM"的技术。

### OkHttp
**一句话**:安卓 App 最常用的网络请求库。
**逆向中**:Hook OkHttp 就能拦截所有请求,是抓包利器。

---

## P

### Packer(加壳器)
**一句话**:把 APK "包一层壳"保护起来的工具。
**常见加固厂商**:360 加固、腾讯乐固、梆梆、爱加密、阿里聚安全、顶象。
**逆向要做什么**:先"脱壳"才能看到真实代码。

### Ptrace
**一句话**:Linux 系统调用,一个进程可以"附加"到另一个进程上调试它。
**检测场景**:App 会用 ptrace 检测自己是否被调试。

### Python
**一句话**:最常用的脚本语言,逆向脚本 90% 用 Python 写。

---

## R

### Root
**一句话**:获取手机的"最高权限"(管理员权限)。
**为什么逆向需要**:很多 Hook 操作要 Root 才能做。
**工具**:Magisk。

### radare2 / Cutter
**一句话**:**免费开源**的逆向工具,命令行为主。Cutter 是它的 GUI 版。

---

## S

### Shell
**一句话**:命令行界面。adb shell 就是进入手机的命令行。

### Signature(签名)
**一句话**:
- **APK 签名**:证明 APK 是谁发布的,防篡改
- **接口签名**:API 请求带的 `sign` 参数,防止别人乱调接口

### Smali
**一句话**:Android 的"汇编语言",是 DEX 代码的人类可读形式。
**类比**:Java 是中文,Smali 是拼音,DEX 是二进制。
**逆向中**:改代码要改 smali,但读代码可以看 jadx 的 Java 伪代码。

### SSL Pinning(证书锁定/固定)
**一句话**:App 只信任自己预设的证书,即使你装了代理证书也抓不到包。
**破解方法**:Frida Hook 绕过。

### SO 文件(.so)
**一句话**:Android 里的"动态链接库",C/C++ 编译的产物。
**类比**:Windows 的 `.dll`。
**为什么重要**:很多核心算法藏在 so 里,Java 层只是个"调用入口"。

### SRC (Security Response Center)
**一句话**:各大公司的"安全应急响应中心",你找到漏洞提交给他们,会拿奖金。
**比如**:阿里 ASRC、腾讯 TSRC。

---

## T

### TracerPid
**一句话**:Linux `/proc/[pid]/status` 里的一个字段,如果非 0 表示被调试。
**App 反调试**会检查这个值。

---

## U

### Unidbg
**一句话**:用 Java 代码"模拟"执行 so 的工具。
**经典场景**:爬虫想调 App 的签名算法,不用装手机,Unidbg 直接跑 so 拿结果。

---

## V

### VMP (Virtual Machine Protection)
**一句话**:把代码编译成**自定义的虚拟机指令**,让逆向极度困难。
**高级保护手段**,大神才玩得转。

---

## X

### Xposed
**一句话**:老牌 Hook 框架,现在基本被 LSPosed 取代。
**特点**:开机自动加载模块,持久化 Hook。

---

## 中文名词

### 脱壳
**一句话**:加固 App 的代码被"壳"保护了,"脱壳"就是把真实代码从内存里取出来。

### 反编译
**一句话**:把编译后的二进制(如 DEX)**反推**回源代码。jadx 干的就是这事。

### 反调试
**一句话**:App 检测自己是否被调试,被调试就崩溃/退出。
**绕过**:Hook 掉检测函数。

### 反模拟器
**一句话**:App 检测自己是否跑在模拟器上,是就拒绝运行。

### 加固 / 加壳
**一句话**:给 APK 套一层保护壳,防逆向。

### 爬虫
**一句话**:自动抓取 App/网站数据的程序。很多人学逆向是为了给 App 写爬虫。

### 混淆
**一句话**:把代码里的变量名、类名换成无意义的字符(`a`、`b`、`c`...),让你看不懂。

### 签名校验
**一句话**:App 检查自己有没有被人重新打包,被改了就闪退。
**绕过**:Hook 签名校验函数,让它永远返回"没被改"。

### 重打包
**一句话**:反编译 → 改代码 → 再打包成 APK → 装机。
**用处**:去广告、破解 VIP 的经典路线。
