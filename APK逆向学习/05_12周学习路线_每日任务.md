# 05 | 12 周学习路线(每周目标 + 每日任务)

> **按这份计划走,12 周后你能达到"初级逆向工程师"水平。**
> 每天 1.5 - 2 小时,每周 1 次实战,全程用 AI 辅助。

---

## 🗓 总览:12 周 4 阶段

| 阶段 | 周次 | 目标 | 毕业标志 |
|------|-----|------|---------|
| **阶段一:地基** | 第 1-3 周 | 编程基础 + Android 基础 | 能读懂 Java 代码 |
| **阶段二:静态** | 第 4-6 周 | 会反编译、会改包 | 能去掉一个 App 的广告 |
| **阶段三:动态** | 第 7-9 周 | 会 Frida、会抓包 | 能 Hook 任意函数抓接口 |
| **阶段四:进阶** | 第 10-12 周 | 会 so 分析 + 对抗 | 能还原简单签名算法 |

---

## 📅 第 1 周:Java 基础速成 + 环境搭建

### 🎯 本周目标
- 能读懂基础 Java 语法(类、方法、变量、if/for)
- 装好所有基础工具

### 📝 每日任务

**Day 1(1.5h)**:
- 看 `00_从零开始` + `01_知识架构`
- AI 提问:"用 10 分钟教会我 Java 的 5 个核心概念"
- 不需要会写 Java,只要能**读懂**

**Day 2(2h)**:
- 装 JDK 17、Python 3.10、Git
- 装 Android Studio,创建一个模拟器
- AI 辅助每一步(报错直接问)

**Day 3(1.5h)**:
- Java 语法 1:类 / 对象 / 方法 / 字段
- AI 给你写 5 段代码,让你解释每段在做什么

**Day 4(1.5h)**:
- Java 语法 2:if / for / while / switch
- 同上,AI 出题你答

**Day 5(1.5h)**:
- Java 语法 3:继承 / 接口 / 注解
- 重点理解 `@Override`、`implements`

**Day 6(2h)**:
- 读一段真实 Android 代码(让 AI 给你一段简单 Activity 代码)
- 逐行写注释

**Day 7(验收)**:
- AI 出 10 道 Java 代码阅读题
- 自测通过率 > 70% 才能进下一周

### ✅ 本周验收
- [ ] Android Studio 模拟器能启动
- [ ] `adb devices` 能看到设备
- [ ] 能看懂 50 行 Java 代码大概在做什么

## 📅 第 2 周:Android 基础 + APK 结构

### 🎯 本周目标
- 懂 APK 文件里都有什么
- 懂安卓 App 的四大组件

### 📝 每日任务

**Day 1**:APK 结构解剖
- 找一个简单 APK(去官方网站下一个小 App)
- 把它改名成 `.zip` 解压,看里面有什么
- AI 讲解每个文件/目录的作用

**Day 2**:AndroidManifest.xml
- 用 Android Studio 打开上面的 Manifest
- AI 教你读懂:`activity`、`permission`、`intent-filter`

**Day 3-4**:四大组件
- Activity(页面)
- Service(后台服务)
- Broadcast Receiver(广播)
- Content Provider(数据共享)
- 每个让 AI 举一个生活例子

**Day 5**:APK 打包签名原理
- v1 / v2 / v3 签名区别
- AI 讲解签名机制

**Day 6**:ADB 深度使用
- 10 个常用命令(AI 给你出题)

**Day 7(实战)**:
- 自己写一个"Hello World"App(AI 手把手教)
- 打包、安装到模拟器

### ✅ 本周验收
- [ ] 能说出 APK 里主要文件的作用
- [ ] 会用 ADB 装 / 卸 / 调试 App

## 📅 第 3 周:Smali 语言基础

### 🎯 本周目标
- 能读懂简单的 smali 代码
- 会小幅修改 smali

### 📝 每日任务

**Day 1**:Smali 是什么

**Day 2-3**:Smali 基础语法
- 寄存器(v0, v1, p0, p1)
- 基本指令(const、move、invoke、return)

**Day 6**:修改 smali 实战

- 把 "if 登录成功" 的条件反转

**Day 7**:重打包
- 用 apktool 解包 → 改 smali → 重打包 → 装机 → 验证

### ✅ 本周验收

- [ ] 能成功重打包一次 APK 并在模拟器装上

## 📅 第 4 周:jadx 深度使用 + 静态分析流程

### 🎯 本周目标
- 精通 jadx 的所有常用功能
- 建立"拿到 APK 该怎么办"的分析流程

### 📝 每日任务

**Day 1**:jadx 装好,熟悉界面
- 打开一个 APK 到处点

**Day 2**:搜字符串定位代码

**Day 3**:追踪函数调用链
- "查找用法"、"跳转到声明"

**Day 4**:看 AndroidManifest 找入口
- 从入口 Activity 一路追到核心逻辑

**Day 5**:导出为 .java 项目
- 方便用 IDE 全局搜索

**Day 6-7(实战)**:
- 随便找一个广告多的小 App
- 用 jadx 定位到广告代码,记录分析过程
- 把分析笔记让 AI review

### ✅ 本周验收
- [ ] 拿到一个 APK,能在 10 分钟内找到主入口
- [ ] 能说出一个 App "登录成功后会跳转到哪里"

## 📅 第 5 周:改包去广告实战

### 🎯 本周目标
- 完整走完一次"去广告"流程

### 📝 每日任务

**Day 1**:目标选择
- 选一个简单的、有开屏广告的 App(官方下载)
- **千万别选大厂的**(加固强,新手搞不定)

**Day 2**:定位广告代码
- 搜关键字("广告"、"Ad"、"banner")
- AI 帮你分析哪里是广告入口

**Day 3**:修改策略
- 三种方案:
  1. 让广告方法直接 `return`(最简单)
  2. 把广告视图 `setVisibility(GONE)`
  3. 注释掉广告 SDK 初始化

**Day 4**:修改 smali
- apktool 解包 → 改 → 重打包

**Day 5**:签名 & 安装
- 用 apksigner 签名
- 装机测试

**Day 6**:失败 → 调试 → 重做
- 这是最正常的,AI 陪你一起排查

**Day 7**:写笔记总结
- 完整记录过程,发到看雪论坛或博客

### ✅ 本周验收
- [ ] 成功去掉一个 App 的广告
- [ ] 写一篇 500 字的分析笔记

### 📝 每日任务

**Day 7**:综合练习

## 📅 第 7 周:Frida 入门

### 🎯 本周目标
- 装好 Frida,跑通第一个 Hook

### 📝 每日任务

**Day 1**:装 Frida
- 电脑:`pip install frida-tools`
- 手机:下载对应架构 frida-server,推到手机
- 遇到问题问 AI

**Day 2**:第一个 Hook(Hello World)
- 随便 Hook 一个 App 的方法,打印调用

**Day 3**:Hook 参数
- 函数被调用时,打印它接收的参数

**Day 4**:Hook 返回值
- 修改返回值(比如让 `isVip()` 永远返回 true)

**Day 5**:Hook 构造函数 / 重载方法
- `$init`、`overload`

**Day 6**:调用 App 内部方法
- 用 Frida 直接调 App 的私有函数

**Day 7(实战)**:
- 找一个带登录的 App
- Hook 登录方法,打印账号密码(当然只 Hook 你自己的账号)

### ✅ 本周验收
- [ ] 成功 Hook 一个方法,看到参数和返回值

## 📅 第 8 周:抓包 + SSL Pinning 绕过

### 🎯 本周目标

### 📝 每日任务

**Day 1**:装 Charles(或 mitmproxy)

**Day 2**:装证书抓 HTTPS

**Day 4-5**:Frida 绕过 SSL Pinning
- 用现成脚本(GitHub 上有 `frida-universal-pinning-bypass`)
- AI 教你改脚本适配新 App

**Day 6**:Objection 一键绕过
- `android sslpinning disable`

**Day 7(实战)**:
- 抓 3 个 App 的登录接口,分析请求参数

## 📅 第 9 周:Frida 进阶 + 协议分析

### 📝 每日任务

**Day 5-6**:签名还原实战

- 推断算法

**Day 7(实战)**:
- 用 Python 复现签名算法
- 验证:你的 Python 算出来 = App 算出来

### ✅ 本周验收
- [ ] 能完整分析一个 App 的签名参数生成逻辑
